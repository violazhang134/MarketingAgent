# 术语表 📖

> 本文档解释 Marketing Agent 项目中的专有名词，帮助开发者和 LLM 快速理解上下文。

---

## 产品概念

### Campaign（营销活动）

一次完整的营销推广活动，从竞品分析到素材生成到实验配置的全流程。一个 Campaign 包含分析数据、策略配置、生成素材和实验结果。

### LaunchPack（发布包）

可直接用于投放的素材包。包含视频脚本、广告文案、Hook 素材、落地页文案和分享激励文案。是 Pillar 1（Distribution-ready Pack）的核心产出物。

### Playbook（策略手册）

记录已验证有效的增长策略。每当一组实验判定胜出，其配置会被沉淀到 Playbook 中供后续复用。

### Next Best Action（下一步动作）

基于数据分析给出的具体行动建议。不是抽象的报告，而是可立即执行的操作（如「关停素材 A」「切换落地页模板」）。

## 四大支柱 (The 4 Pillars)

| 支柱         | 英文                    | 核心价值               |
| ------------ | ----------------------- | ---------------------- |
| **Pillar 1** | Distribution-ready Pack | 一键生成可发布的素材包 |
| **Pillar 2** | Experiment Runner       | 自动化 A/B 测试配置    |
| **Pillar 3** | Telemetry & Attribution | 统一埋点与归因         |

## 智能体角色

### Markie（马基）

居住在侧边栏的高维精灵，是用户与系统对话的唯一入口。负责意图识别、策略规划、任务分发和最终验收。性格智慧、冷静，偶尔幽默。

### Minions（小队 / 动物帮）

居住在画布上的像素动物群，负责执行任务。当 Markie 下达指令时，它们会出现在画布上「施工」，为静态节点带来生命感。

### Byte（比特）

**机器猫** - 数据分析专家。蓝色金属外壳，尾巴是数据线。极客性格，专注于数据处理、爬虫、分析任务。工作时头顶会浮现「数据流」。

### Bit（比特犬）

**数据狗** - 创意生成专家。戴着厚厚眼镜，背着小书包。热情忠诚，容易极度兴奋。负责文案写作、创意素材生成。工作时身旁有迷你画板。

### Glitch（故障）

**故障兔** - Debug 专家。身体偶尔像素闪烁。神经质、敏锐，总是担心出错。负责异常处理、风控、质量检查。工作时拿着放大镜或扳手。

## 画布概念

### InfiniteCanvas（无限画布）

可无限平移和缩放的工作区，用于可视化工作流。节点和连接线构成知识图谱，Minions 在其中「施工」。

### CanvasNode（画布节点）

画布上的信息卡片。类型包括：

- `analysis` - 分析结果
- `insight` - 洞察报告
- `creative` - 创意素材
- `experiment` - 实验配置
- `agent_step` - Agent 工作步骤

### CanvasEdge（画布连线）

节点之间的连接关系，表示数据流或逻辑关联。支持流动动画效果。

### ViewMode（视图模式）

画布展示模式：

- `simplified` - 简略模式：每个节点显示标题和摘要
- `detailed` - 详细模式：节点内渲染完整内容

### Viewport（视口）

画布的可视区域。包含 x/y 偏移和 zoom 缩放比例。

## 创意策略

### Hook（钩子）

广告开头用于吸引注意力的内容。常见类型：

- `challenge` - 挑战类（如「我打赌你过不了…」）
- `suspense` - 悬念类（如「你不会相信接下来…」）
- `satisfaction` - 满足类（如「这个音效太解压了」）
- `contrast` - 对比类（如「我妈 vs 我爸」）

### CTA（Call to Action）

行动呼吁，引导用户下载/点击的文案。强度分为 soft/medium/strong。

### CreativeStrategy（创意策略）

创意生成的配置参数，包括 Hook 风格、视觉基调、CTA 强度、目标受众。

## 实验相关

### ExperimentPack（实验包）

A/B 测试的完整配置。包含实验 ID、实验变量、对照组/测试组配置、流量分配比例。

### ExperimentArm（实验组）

实验中的一个变体。包含组名、配置参数和专属追踪链接。

### Variable（实验变量）

实验测试的维度：

- `cover` - 封面图 A/B
- `incentive` - 激励结构 A/B
- `entry` - 入口形式 A/B

## 技术术语

### Store

Zustand 状态管理单元。项目中有三个 Store：

- `useAppStore` - 全局 UI 状态
- `useCampaignStore` - 营销活动状态
- `useCanvasStore` - 画布状态

### Engine

无状态的业务计算引擎。只接收输入、返回输出，不产生副作用。

### GlassCard

毛玻璃风格的卡片组件，是系统的基础 UI 原子。

### Sentient UX（感知体验）

通过音效、动画、微交互让 AI 产品具备「呼吸感」和「生命感」的设计理念。
